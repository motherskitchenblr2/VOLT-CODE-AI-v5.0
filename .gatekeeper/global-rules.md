# Global Rules

These rules apply to all code changes across the project.

1. Use clear, consistent naming conventions.
2. Prefer lowercase for primitive type keywords in TypeScript/JavaScript (e.g., `string`, `number`, `boolean`).
3. Do not use single-letter variable names like `t` unless in very small/local loop scopes.
4. Add JSDoc comments to all exported functions, classes, and public utilities.
5. Keep functions small and focused on one responsibility.
6. Avoid `any` unless absolutely necessary; prefer explicit typing.
7. Write readable code first; optimize only when there is clear need.
8. Preserve existing project structure and conventions.
9. Include basic error handling for all async or external operations.
10. Keep imports organized and remove unused code.
